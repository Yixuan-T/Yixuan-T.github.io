类和对象设计

面向对象基础

面向对象程序设计的四个主要特征:

  • 抽象和具体：对应了类与对象
  • 封装
  • 继承
  • 多态

类和对象

  • 类是描述对象的“基本原型”，它定义一种对象所能拥有的属性和能完成的操作 。

    对象，是类的一个实例。

封装

封装是将数据和方法捆绑在一起，并控制外部对其修改和访问的程度。

继承

继承是对一个类的扩充或修正所定义的一个类.

  • 通过继承，子类可以利用父类中定义的方法和属性，就像它们属于子类本身一样.

多态

多态（Polymorphism）指类中同一名称的方法可以有多种不同的功能，多态的实现包括：

  • 重载overload
  • 覆盖override

方法的重载(编译时多态性)：在同一个类中有多个方法用同一个名字，但有不同的参数。

方法的覆盖（运行时多态性）：在子类中重新定义父类中已有的方法，又称方法重写。

java类定义

类定义包括类声明和类体
类体当中包括成员变量、构造方法、成员方法。
类的基本定义：

方法体和方法重载

  • 方法定义包括方法声明和方法体

  • 方法体包括所有合法Java代码及局部变量声明。
    局部变量的作用域在该方法内部。若局部变量与类的成员变量同名，则类的成员变量被隐藏。此时要访问该成员变量使用this.变量名访问。
    成员方法若无返回值，则返回值类型为void

  • 方法重载指多个方法享有相同的方法名
    区别在于：参数类型、或个数不同；

构造方法

构造方法用于将类实例化成一个对象
构造方法具有和类名相同的名称，无需声明返回类型
构造方法只能由new运算符调用
构造方法也可以重载

对象实例化和使用

对象声明：ClassName reference
例，Car car;
声明并不为对象分配内存空间，而只产生一个引用；

对象实例化：运算符new为对象分配内存空间，并调用对象的构造方法进行初始化，返回引用。
例，car = new Car(3); Car o = new Car();
对象引用指向对象所在堆内存地址。

通过运算符“.”可调用对象的成员变量和方法
访问成员变量：reference.variable；例，car.color
调用成员方法： reference.method([paramlist]);例，car.addOil();

对象的清除

当不存在对一个对象的引用时，该对象成为一个无用对象，即“垃圾”。Java的垃圾回收机制会自动进行垃圾的清除和内存的回收。

System.gc( );函数建议Java虚拟机尽快开展垃圾回收任务。
若类实现了finalize()方法，则该方法在垃圾收集前会被自动调用

  • 类似C++的析构函数，但并不如C++中的常用

参数传递

Java中，调用函数时，参数被复制传递到该函数中。

static关键字

成员变量定义时加上static修饰符，成为静态变量

  • 定义：static type variable;
  • 静态变量是类的一部分，与具体的对象无关。
  • 可通过类名.静态成员直接访问，也可以通过实例对象引用名.静态成员访问类变量

成员方法定义时加上static修饰符，成为静态方法

static returnType classMethod({paramlist]){…}
静态方法不能访问普通的成员变量，只能访问静态变量
静态方法可由类名.静态方法直接调用.
静态方法中不能使用this。

封装性－类、变量和方法的修饰

类前面的访问修饰符是：public或者 “无”

  • public类不但可以被同一包中的其它类使用,别的包中的类也可以使用.
  • 缺省：类只能被本包中的其它类使用
  • 一个Java源文件中只能有一个public类，且该类的名称与文件名必须一样。

变量和方法的修饰符public、protected、private、缺省。

  • 用以控制外部类对变量、方法的可访问程度。
  • 每个类内部的成员方法和成员变量可以知道彼此的存在，可以相互使用，不论public或private修饰

*public修饰: *任何其它类、对象只要可以看到这个类的话，那么它就可以存取变量的数据，或使用方法。

private修饰:不允许任何外部类存取和调用

缺省（前边没有修饰字的情况）:在同一包中出现的类才可以直接使用它的数据和方法.

protected修饰:如果变量或方法有修饰字protected，同一包中的类、该类的子类可以使用。

封装性修饰符的使用原则：尽量使用访问范围小的。
