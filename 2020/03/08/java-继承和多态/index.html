继承与多态

继承

继承是对一个类的扩充或修正所定义的一个类.

通过继承，子类可以利用父类中定义的方法和属性，就像它们属于子类本身一样.

Java继承的语法：关键字extends

通过继承实现代码复用，并维护类之间的关系,子类可以重写父类的方法.

Java中若不设置父类，则默认继承Object。所有类的根类为java.lang.Object；

Java不支持多重继承，单重继承结构更清晰，多重继承功能上的缺失可通过接口Interface弥补

子类在构造时，首先会调用父类的构造函数（相当于默认加上了super()语句），这个过程是迭代的；所以Java中，任何对象在初始化时，首先调用java.lang.Object的构造函数。

instanceof关键字

instanceof：对象是否是这个特定类或者是它的子类的一个实例，返回true 或 false。

对象引用的类型转换

  • 一个类的对象可以被该类的引用，或者该类的父类的引用所指向。

    对象有对象的类型，即对象实例化时的类型，如new Employee();
    引用有引用的类型，即引用声明时的类型，如Employee e;

  • 引用向上的类型转换是自动实现的。
    把引用的类型从子类转换成父类；表示所指对象暴露出功能的减少。

  • 引用向下的类型转换必须通过强制转换。
    把引用的类型从父类转换成子类；表示所指对象暴露出功能的增加。
    若转换失败，则在运行时扔出ClassCastException异常。为避免强制转换失败，可通过instanceof先进行判断。

super关键字

super用来引用当前对象的父类

  • 访问父类中因同名而被隐藏的成员变量，如：
    super.variable;
  • 调用父类中被重写的方法，如：
    super.Method([paramlist]);
  • 调用父类的构造函数，如：
    super([paramlist]);

多态性与方法重写(override)

  • 多态性是由封装性和继承性引出的面向对象程序设计语言的一个主要特征。

    静态多态性（编译时多态）：由方法重载(overload)实现
    动态多态性（运行时多态）：由方法重写(override)实现，调用规则：根据对象的类型决定调用的方法。

  • 方法重写应遵循的原则：

子类中重写的方法和父类中被重写的方法要具有相同的名字，相同的参数列表和相同的返回类型

重写后的方法不能比被重写的方法有更严格的访问权限

重写后的方法不能比被重写的方法产生更多的异常

final关键字

final修饰成员变量，则该变量不能修改，成为常量

final修饰成员方法，则该方法不能被子类重写

final修饰类，则类不能被继承

类java.lang.Object

Object类是所有Java类的祖先，定义类时不指定父类则默认父类为Object。其定义的方法包括：

  • boolean equals(Object o)
    判断两个引用是否指向同一个对象，若相同返回true，否则false。相当于操作符“==”。

  • String toString()
    返回对象的字符串表示，默认的为类名 + hashcode

  • native int hashCode();
    返回对象的哈希值，Object类中hashcode方法返回的哈希值与对象的存储地址有关。
    native关键字表示该方法的实现依赖于本地代码，如c/c++。

  • final native Class getClass()
    返回该对象的类对象
    final表示不可被重写

  • final native notify(), notifyAll(), wait()
    用于多线程处理中的同步。

  • void finalize()
    用于在垃圾回收前完成相应资源回收。

Class对象（类对象）
“万事万物皆对象，类也是对象”：类对象描述类定义中所有信息，如构造方法、成员方法、成员变量、包等信息。
一个类仅当被装载时，它的类对象才由JVM自动生成。比如，类Point的类对象，在Point类被JVM装载时构造生成。

常用方法：
返回完整类名：String getName()
返回公有成员：getFields,getMethods,getConstructors
返回所有成员: getDeclaredFields,getDeclaredMethods,getDeclaredConstructors
类对象是反射功能实现的基础：forName(String className), Object newInstance()

抽象类与抽象方法

abstract修饰的类与方法称为抽象类与抽象方法

  • 抽象方法只有声明，没有实现；
  • 若类中包含了抽象方法，则该类必须被定义为抽象类
  • 抽象类不能被实例化，其子类若实现了所有抽象方法，则可被实例化。
  • 抽象类只能作为引用的类型

接口

接口只包含常量和方法的定义，而没有变量和方法的实现（JDK8之后接口中可实现静态方法）

接口声明：[public] interface 接口名[extends 父接口]{}
接口体包括常量定义和方法定义
常量定义: type NAME=value; 该常量被实现该接口的所有类共享; 直接具有public, final, static的属性.
方法直接具有public和abstract属性

接口的实现

  • 在类声明中用implements关键词表示该类实现某个接口。
  • 一个类可实现多个接口：在implements后用逗号分开
  • 必须实现接口中定义的所有方法，否则该类只能是抽象类

接口的作用：

  • 接口可被视为Java对于多重继承功能的一种变相实现。
  • 通过接口实现不相关类的相同行为,无需考虑继承关系
  • 接口可作为引用类型：Collection c = new Queue();

接口与抽象类异同

相同点
一般都包含抽象方法

都不能直接实例化，只能用作引用类型

区别
类只能继承extends一个抽象类，但可实现implements多个接口

接口只能定义抽象方法（JDK 8可以定义静态方法），抽象类可以有部分方法有具体的实现（可视为一种半成品）。

接口中成员属性只能定义成常量，而抽象类则不同

内部类

在一个类的内部定义的类，称为内部类。

  • 成员内部类
    作为外部类的成员，和成员方法、成员变量并列；

  • 局部内部类
    在方法当中定义的内部类。
    匿名内部类是一种特殊的局部内部类

  • 静态内部类（又称嵌套类）
    当成员内部类有static修饰时

成员内部类、局部内部类、静态内部类之间的区别与成员变量、局部变量、静态变量间的区别一样。

  • 成员内部类、局部内部类和匿名内部类

可使用外部类的所有成员变量和方法，即使是private的；若内部类处于方法当中，则还可使用该方法中的final局部变量。

成员内部类、局部内部类初始化时，要使用外部类对象的引用.new 内部类的格式，如reference.new InnerClass()

  • 静态内部类
    不能使用成员变量和方法，这点同静态方法、变量。

初始化时，使用new 外部类类名.内部类类名的格式，如new OuterClass.InnerClass()

  • 匿名类
    在一个表达式内部包含一个完整的类定义，没有类名
    定义时需要继承某个类或实现某个接口
    只能创建一个对象，它通常用来简化代码编写
